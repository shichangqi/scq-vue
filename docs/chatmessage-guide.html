ChatMessage 组件使用指南

ChatMessage 用来呈现聊天产品中的单条消息。它负责把接口返回的文本、结构化数据、Markdown、媒体和附件展示成稳定的消息气泡，并提供状态、流式输出、图片预览、附件点击等业务接口。

适用版本scq-vue 1.0.16 及以上

1. 组件定位

ChatMessage 只处理“单条消息”的展示和交互，不内置消息列表、滚动到底部、接口请求或会话状态管理。这样做的好处是边界清楚：业务侧可以自由组合消息列表、SSE、WebSocket、分页加载、消息撤回等逻辑。

  • AI / 用户气泡
  • Markdown
  • JSON
  • 代码高亮
  • 图片预览
  • 视频播放
  • 文件附件
  • 流式状态
  • 状态提示
  • 操作插槽
典型使用场景：AI 助手、客服对话、知识库问答、工单沟通、审核记录、数据分析结果展示、带附件的消息流。

2. 安装与引入

2.1 安装

pnpm add scq-vue
# 或
npm install scq-vue
# 或
yarn add scq-vue

2.2 全局注册

适合中后台或组件使用频率较高的项目。样式可直接引入完整样式包。

import { createApp } from 'vue'
import ScqVue from 'scq-vue'
import 'scq-vue/style.css'
import App from './App.vue'

createApp(App).use(ScqVue).mount('#app')
<template>
  <scq-chat-message message="你好，我是 SCQ Assistant。" role="ai" />
</template>

2.3 按需引入

如果项目只用到 ChatMessage，可以按需引入组件和对应样式。

import { ChatMessage } from 'scq-vue'
import 'scq-vue/styles/chat-message.css'
<template>
  <ChatMessage :message="message" role="ai" />
</template>

<script setup lang="ts">
import { ChatMessage } from 'scq-vue'
import 'scq-vue/styles/chat-message.css'

const message = '这是一条来自接口的普通文本。'
</script>
样式说明：使用 `scq-vue/style.css` 会引入构建后的完整样式；使用 `scq-vue/styles/chat-message.css` 只引入 ChatMessage 的独立样式。团队可以根据打包策略选择其中一种。

3. 快速上手

最少只需要传入 `message`。`role` 决定气泡方向和视觉样式，默认是 `ai`。

<template>
  <scq-chat-message
    message="我已经整理好结果，你可以继续追问。"
    role="ai"
    name="SCQ Assistant"
    :timestamp="Date.now()"
  />

  <scq-chat-message
    message="帮我把结论压缩成三句话。"
    role="user"
    name="Chauncy"
    :show-time="false"
  />
</template>

如果要做消息列表，建议由外层组件负责循环、滚动和数据加载，ChatMessage 只负责渲染一条消息。

<template>
  <section class="message-list">
    <scq-chat-message
      v-for="item in messages"
      :key="item.id"
      :message="item.content"
      :role="item.role"
      :name="item.name"
      :timestamp="item.createdAt"
    />
  </section>
</template>

4. 内容类型

`contentType` 控制消息正文的渲染方式。默认值是 `auto`，组件会先识别媒体，再识别 JSON 和 Markdown，最后回退到普通文本。

类型适合场景说明
auto接口返回内容不固定自动识别图片、视频、JSON、Markdown 和纯文本。
text普通聊天文本按纯文本显示，保留换行，不解析 Markdown。
markdownAI 回复、富文本说明、代码片段支持标题、列表、链接、表格、代码块和自动换行。
json调试接口、展示结构化结果对象会格式化成缩进 JSON，字符串 JSON 会尝试美化。
image图片消息、截图、生成图支持懒加载、加载失败文案和点击预览。
video视频消息、录屏、演示片段支持 controls、poster、autoplay、muted、loop、preload。

4.1 普通文本

<scq-chat-message
  content-type="text"
  message="第一行\n第二行会保留换行。"
/>

4.2 Markdown 与代码高亮

适合 AI 输出、使用说明和带代码的回答。Markdown 链接默认添加 `target="_blank"` 和 `rel="noopener noreferrer"`，也可以通过配置修改。

<scq-chat-message
  role="ai"
  content-type="markdown"
  :message="markdownText"
  link-target="_blank"
  link-rel="noopener noreferrer"
/>
const markdownText = `# 分析结果

- 已读取项目配置
- 已完成构建检查

\`\`\`ts
const passed = true
\`\`\``

4.3 JSON 数据

<scq-chat-message
  role="ai"
  content-type="json"
  :message="apiResult"
/>
const apiResult = {
  code: 0,
  message: 'ok',
  data: [{ id: 1, name: 'SCQ' }],
}

5. 头像、昵称和时间

`avatar`、`avatarText`、`avatarAlt`、`name` 和 `timestamp` 用于补充消息来源。头像图片、头像文字和展示昵称可以分别配置；没有头像图片时，会优先显示 `avatarText`，未传时再使用昵称首字母。

<scq-chat-message
  message="头像可以传图片，也可以只传昵称。"
  role="ai"
  name="SCQ Assistant"
  avatar-text="AI"
  avatar-alt="SCQ Assistant avatar"
  avatar="https://example.com/avatar.png"
  :timestamp="Date.now()"
  :time-formatter="formatTime"
/>
const formatTime = (value: string | number | Date | null | undefined) => {
  if (!value) return ''
  const date = value instanceof Date ? value : new Date(value)
  return date.toLocaleString('zh-CN', { hour12: false })
}
需求推荐配置
紧凑消息列表`showAvatar=false`、`showName=false`、`showTime=false`
客服或多人会话保留 `name` 和 `avatar`，便于区分角色。
消息时间格式统一在外层封装 `timeFormatter`，不要在每条消息里写重复逻辑。

6. 流式输出

`streaming` 用于标记消息处于流式输出状态，不负责建立 SSE 或 WebSocket 连接，也不再额外显示光标。业务侧持续更新 `message` 即可。

<template>
  <scq-chat-message
    :message="answer || '正在生成...'"
    role="ai"
    content-type="markdown"
    :streaming="streaming"
  />
</template>

<script setup lang="ts">
import { ref } from 'vue'

const answer = ref('')
const streaming = ref(false)

const start = () => {
  const source = new EventSource('/api/chat')
  streaming.value = true

  source.onmessage = (event) => {
    answer.value += event.data
  }

  source.onerror = () => {
    streaming.value = false
    source.close()
  }
}
</script>
行为说明：流式输出时组件会保留 `aria-live` 提示，方便辅助技术感知内容更新。

7. 消息状态

状态适合展示“正在思考”“正在调用工具”“处理完成”“失败”等消息级反馈。组件统一通过 status 对象配置状态；不传 status 时，不展示状态区域。

<scq-chat-message
  message="正在整理上下文，稍后给出完整回答。"
  role="ai"
  :status="{ type: 'thinking', text: '正在思考中' }"
/>

<scq-chat-message
  message="构建检查完成。"
  role="ai"
  :status="{ type: 'success', text: '处理完成', loading: false }"
/>
状态类型适合场景
thinkingAI 正在组织回答、规划步骤。
tool正在调用接口、工具、插件或外部系统。
loading普通处理中状态。
success任务已经完成。
warning结果可用，但需要用户注意。
error请求失败、工具失败或内容不可用。

8. 图片与视频

8.1 图片消息

图片可以直接传 URL，也可以传对象。`previewable` 控制是否允许点击预览，加载失败会显示 `imageFallbackText`。

<scq-chat-message
  :message="imageUrl"
  role="ai"
  content-type="image"
  image-fallback-text="图片暂时无法加载"
  @preview-open="handlePreviewOpen"
  @preview-close="handlePreviewClose"
  @image-error="handleImageError"
/>
const imageUrl = 'https://example.com/result.png'

const handlePreviewOpen = (src: string) => {
  console.log('打开预览', src)
}

const handlePreviewClose = (src: string) => {
  console.log('关闭预览', src)
}

const handleImageError = (src: string) => {
  console.warn('图片加载失败', src)
}

8.2 视频消息

<scq-chat-message
  role="user"
  content-type="video"
  :message="videoPayload"
/>
const videoPayload = {
  src: 'https://example.com/demo.mp4',
  poster: 'https://example.com/poster.png',
  controls: true,
  preload: 'metadata',
}

8.3 自动识别媒体

`contentType="auto"` 时，组件会识别对象里的 `type`、`mediaType`、`contentType`，也会根据 URL 后缀判断图片或视频。

const mediaMessage = {
  type: 'image',
  src: 'https://example.com/chart.webp',
  alt: '分析图表',
}

9. 文件附件

附件适合展示模型生成的文件、用户上传的材料、工单附件和下载资源。组件会根据文件名识别常见类型，也可以手动指定 `type`。

<scq-chat-message
  :message="attachmentMessage"
  role="ai"
  :attachments="fileAttachments"
  :attachment-click="handleAttachmentClick"
  @attachment-click="trackAttachmentClick"
/>
const attachmentMessage = '下面是本次分析用到的文件。'

const fileAttachments = [
  {
    name: '需求说明.pdf',
    url: 'https://example.com/spec.pdf',
    size: 245760,
    download: true,
  },
  {
    name: '数据分析.xlsx',
    url: 'https://example.com/report.xlsx',
    size: '1.2 MB',
  },
  {
    name: '上传中.zip',
    status: 'uploading',
    progress: 48,
    statusText: '上传中',
  },
]

const handleAttachmentClick = (payload, event) => {
  if (payload.status === 'uploading') {
    event.preventDefault()
    return false
  }
}

const trackAttachmentClick = (payload) => {
  console.log('附件点击', payload.name, payload.type)
}
附件状态展示含义常见用法
default普通附件可打开或下载。
uploading上传中配合 `progress` 显示进度，默认阻止打开。
success上传或生成成功用于明确任务结果。
error失败提示用户重新上传或重试。

10. 消息操作

通过 `actions` 插槽可以放置复制、重新生成、点赞、点踩、引用、删除等按钮。组件只提供容器样式，具体行为由业务决定。

<scq-chat-message message="这是一条可操作的回答。" role="ai">
  <template #actions>
    <button type="button" class="scq-chat-message__action" @click="copy">复制</button>
    <button type="button" class="scq-chat-message__action" @click="retry">重新生成</button>
    <button type="button" class="scq-chat-message__action" @click="feedback('up')">有帮助</button>
  </template>
</scq-chat-message>
建议：如果同一项目中所有消息都有相同操作，可以在业务侧再封装一层 `AppChatMessage`，把复制、重试、反馈等动作统一进去。

11. 样式定制

ChatMessage 的样式类名统一使用 scq-chat-message 前缀。项目里如果需要调整消息框外观，推荐在业务样式文件中覆盖这些类名，而不是直接改组件源码。这样后续升级组件时更容易合并。

11.1 推荐覆盖方式

全局样式文件中可以直接覆盖类名。若在 Vue 单文件组件的 scoped 样式里修改，需要使用 :deep() 穿透到组件内部。

/* src/styles/chat-theme.css */
.scq-chat-message.is-ai .scq-chat-message__bubble {
  background: #ffffff;
  border-color: #d7e2f0;
  color: #1f2937;
}

.scq-chat-message.is-user .scq-chat-message__bubble {
  background: #1677ff;
  border-color: #1677ff;
  color: #ffffff;
}

.scq-chat-message__content {
  max-width: min(86%, 760px);
}
<style scoped>
.chat-page :deep(.scq-chat-message.is-ai .scq-chat-message__bubble) {
  background: #ffffff;
}

.chat-page :deep(.scq-chat-message__avatar) {
  width: 40px;
  height: 40px;
  flex-basis: 40px;
}
</style>

11.2 常用样式入口

类名可调整内容常见场景
.scq-chat-message单条消息的外层间距、整体布局。调整消息之间的距离。
.scq-chat-message.is-aiAI 消息整体方向和派生样式。给助手消息设置统一主题。
.scq-chat-message.is-user用户消息整体方向和派生样式。修改用户气泡颜色或对齐方式。
.scq-chat-message__body头像和内容区的横向布局。调整头像与气泡的间距。
.scq-chat-message__content消息内容最大宽度、内部排列。让气泡更宽或更窄。
.scq-chat-message__bubble气泡背景、边框、圆角、阴影、字号、内边距。定制消息框视觉风格。
.scq-chat-message__avatar头像尺寸、边框、背景。适配项目头像尺寸规范。
.scq-chat-message__name昵称字号、颜色、间距。多人会话或客服场景。
.scq-chat-message__time时间颜色、字号、位置。弱化时间信息或贴近气泡。
.scq-chat-message__markdownMarkdown 段落、表格、链接、代码样式。让 AI 输出符合项目内容样式。
.scq-chat-message__code-block代码块背景、文字颜色、滚动条、圆角。适配深色或浅色代码主题。
.scq-chat-message__attachments附件列表宽度和间距。调整附件卡片展示密度。
.scq-chat-message__attachment附件卡片背景、边框、悬停效果。贴合项目文件卡片风格。
.scq-chat-message__actions操作区布局和间距。放置复制、重试、反馈按钮。

13. 使用建议

  • 消息列表、滚动到底部、接口请求和重试队列放在外层组件里管理，ChatMessage 保持单条消息职责。
  • Markdown 内容来自用户输入时，当前组件默认关闭 HTML 解析；如果业务要放开 HTML，需要另外做安全审核。
  • 附件点击如果要接入权限校验、预览系统或下载签名，优先使用 `attachmentClick` 接管默认行为。
  • 图片预览事件适合做埋点、暂停页面快捷键、恢复滚动锁定等工作。
  • 复杂项目建议封装业务版消息组件，把当前用户信息、统一时间格式、默认操作区、埋点等逻辑收进去。
注意：ChatMessage 不是聊天 SDK。它不负责网络连接、消息协议、登录态和会话持久化。把这些职责留在业务层，组件会更容易维护，也更方便在不同产品里复用。

更多组件示例和在线演示可参考：https://shichangqi.github.io/scq-vue/#/guide
